detect_cycles() — DFS con pila de recursión 
def detect_cycles(self):
    visited  = set()   # nodos ya procesados
    rec_stack = set()  # nodos en el camino actual
    path     = []       # camino para reconstruir el ciclo

    def dfs(u):
        visited.add(u)
        rec_stack.add(u)
        path.append(u)

        for v in self.graph.get(u, []):
            if v in rec_stack:
                # v ya está en el camino actual → CICLO
                idx = path.index(v)
                new_cycle = path[idx:] + [v]
                self.cycles.append(new_cycle)

            elif v not in visited:
                dfs(v)  # explorar más profundo

        rec_stack.remove(u)  # salir del camino actual
        path.pop()

    for node in self.graph:
        if node not in visited:
            dfs(node)

# Resultado para nuestra app:

Explicación
Cómo funciona el analizador
01 / GRAFO
Construir el grafo
Cada módulo es un nodo. Cada import es una arista dirigida. Se almacena como diccionario { módulo: [deps] }.
app/page.tsx
→ HomeScreen
→ CartScreen
→ OrderScreen
02 / DFS
Detectar ciclos
Se usa DFS con una pila de recursión. Si durante la exploración encontramos un nodo que ya está en el camino actual, hay un ciclo.
useCart → useOrder
useOrder → useCart
⚠ CICLO detectado
03 / OUTPUT
Generar entregables
Produce un .json con el reporte completo y un .dot para Graphviz con nodos críticos en rojo.

→ dependency_graph.dot
